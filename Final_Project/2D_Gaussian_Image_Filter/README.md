# Final Project вЂ“ 2D Gaussian Image Filter

This folder contains the final project for **Digital System Design with HDL (CE213)**.

The project implements and verifies a hardware-based **2D Gaussian image filter** using Verilog HDL. The design supports image filtering with Gaussian kernels and uses the Lena image as the main test image to compare original, noisy, and processed outputs.

## Main Components

### Verilog HDL Source Code

The hardware design is organized into several RTL modules:

- `Top_module.v`: top-level module of the image filtering system.
- `Controller.v`: control unit for managing processing states.
- `Datapath.v`: datapath for image processing operations.
- `Core.v`: core filtering computation block.
- `Line_Buffer.v`: line buffer used to store neighboring image rows.
- `BRam.v`: block RAM module for image data storage.
- `Tb_top_module.v`: testbench for simulation and functional verification.

### Python Support Scripts

Python scripts are used for image preparation, noise generation, conversion, and verification:

- `source_code/python/src/Noise_Image.py`: generates noisy images.
- `source_code/python/src/matlab.py`: prepares or converts image data for testing.
- `source_code/python/src/Kiemthu.py`: verifies and visualizes output results.

## Image Data

### Original Images

The `images/original/` folder stores the Lena input image before noise is added or filtering is applied.

### Noisy Images

The `images/noisy/` folder stores the Lena image after noise is added. This image is used as input for the Gaussian filter.

### Processed Images

The `images/processed/` folder stores filtered Lena output images generated by hardware simulation or software verification.

---

- `images/original/Lena_image.jpg`: original Lena image.
- `images/noisy/noisy_image.jpg`: noisy Lena image.

- `images/processed/image_out_5x5.jpg`: Lena image after 5 Г— 5 Gaussian filtering.

## HEX Data

The `data_hex/` folder contains image data in HEX format for simulation and comparison:

- `data_hex/input/`: original input image data.
- `data_hex/noisy/`: noisy image data.
- `data_hex/output/`: filtered output image data.
